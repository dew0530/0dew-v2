# 김이슬 작가 공식 홈페이지 — 관리 설명서

이 홈페이지는 글·사진·영상 정보를 전부 `data/` 폴더의 JSON 파일에서 불러옵니다.
**개발 지식이 없어도 괜찮습니다.** 두 가지 방법 중 편한 쪽으로 관리하세요.

| 방법 | 추천 대상 |
|---|---|
| ① **admin.html 관리 페이지** (권장) | JSON을 전혀 몰라도 됨. 입력 칸에 쓰고 버튼만 누르면 올바른 파일이 만들어짐 |
| ② JSON 파일 직접 수정 | 조금 익숙해진 뒤, 빠르게 고치고 싶을 때 |

---

## 1. admin.html 관리 페이지 사용법 ⭐

배포된 주소 뒤에 `/admin.html`을 붙여 접속하세요.
예: `https://0dew0dew.pages.dev/admin.html`

화면 위 탭: **📚 책 · 🎤 강연 · 📸 사진 · 📰 소식 · 💬 후기 · ❓ FAQ · ⚙️ 기본 정보 · 🩺 파일 검사**

### 수정은 언제나 같은 3단계
1. 탭에서 내용을 고치거나 [＋ 추가] / [✕ 삭제] / [↑↓ 순서]를 사용합니다.
2. **[💾 ○○.json 내려받기]** 버튼을 누릅니다. → 올바른 문법의 파일이 자동으로 만들어져 내려받아집니다.
3. 내려받은 파일을 홈페이지 폴더의 `data/` 안에 **같은 이름 파일과 바꿔치기**한 뒤, 폴더를 다시 배포합니다.

> ⚠️ 관리 페이지는 실제 사이트를 바로 바꾸지 않습니다. **내려받기 → 교체 → 재배포**까지 해야 반영됩니다.
> (그래서 다른 사람이 admin.html에 들어와도 사이트를 망가뜨릴 수 없어요.)

> 💡 사진 파일 자체는 관리 페이지로 올릴 수 없습니다. 새 사진은 먼저 `images/` 폴더에 넣고,
> 관리 페이지에는 그 **경로**(예: `images/gallery-banchan-9.jpg`)만 적습니다.

### 🩺 파일 검사 탭
JSON을 직접 고치다가 걱정될 때 사용하세요. 파일을 선택하거나 내용을 붙여넣고 [검사하기]를 누르면
✅ 통과 여부와, 틀렸을 경우 **몇 번째 줄의 어디가 잘못됐는지 ▲ 표시**로 짚어 줍니다.
(사이트 화면에서도 JSON이 깨지면 위쪽에 "몇 번째 줄 근처 오류" 안내 띠가 자동으로 나타납니다.)

## 1-1. GitHub 연결하기 — 버튼 하나로 저장·배포 🚀 (한 번만 설정)

연결하면 admin.html에서 **[🚀 사이트에 올리기] 버튼 하나로**: 저장 → GitHub 커밋 → Cloudflare 자동 배포가 이루어지고,
사진·PDF도 [올리기] 버튼으로 폴더에 바로 업로드됩니다. (연결 전에는 기존 내려받기 방식이 그대로 동작합니다.)

### 1단계. GitHub 계정·저장소 만들기
1. https://github.com 에서 무료 가입 (이미 있다면 로그인)
2. 오른쪽 위 ➕ → **New repository** → 이름 예: `kimyiseul-site` → **Private** 선택 → Create repository
3. 만들어진 저장소 화면에서 **uploading an existing file** 링크 클릭 → 홈페이지 폴더 안의 **모든 파일과 폴더를 드래그**해서 올리고 → Commit changes
   (한 번에 안 올라가면 images, pdfs 폴더를 나눠서 올려도 됩니다)

### 2단계. Cloudflare Pages를 GitHub과 연결하기
1. https://dash.cloudflare.com → Workers & Pages → **Create** → Pages → **Connect to Git**
2. GitHub 로그인 후 방금 만든 저장소 선택 → 빌드 설정은 **전부 비워 둔 채**(Framework: None) → Save and Deploy
3. 이제부터는 GitHub에 파일이 올라올 때마다 Cloudflare가 **1~2분 안에 자동으로 배포**합니다.
   ※ 기존에 드래그 앤 드롭으로 만든 프로젝트는 Git 연결로 바꿀 수 없어서, 이렇게 **새 프로젝트를 만들고**
   기존 주소(커스텀 도메인)가 있다면 새 프로젝트로 옮겨 주세요. 임시 주소(xxx.pages.dev)는 새로 발급됩니다.

### 3단계. 액세스 토큰(비밀번호 같은 열쇠) 만들기
1. GitHub 오른쪽 위 프로필 → **Settings** → 왼쪽 맨 아래 **Developer settings**
2. **Personal access tokens → Fine-grained tokens → Generate new token**
3. 이렇게 설정하세요:
   - Token name: `홈페이지 관리` (아무거나)
   - Expiration: 1년 (만료되면 다시 발급해서 바꿔 넣으면 됩니다)
   - Repository access: **Only select repositories** → `kimyiseul-site` 선택
   - Permissions → Repository permissions → **Contents: Read and write** (이것 하나만!)
4. Generate token → `github_pat_...`로 시작하는 긴 글자를 복사 (이 화면을 벗어나면 다시 볼 수 없어요)

### 4단계. admin.html에 입력
admin.html 위쪽 **[🔗 GitHub 연결]** 상자를 열고 아이디, 저장소 이름, 브랜치(main), 토큰을 넣고
[연결 저장] → [연결 테스트] → ✅가 뜨면 끝!

> 🔐 **토큰 보안**: 토큰은 지금 사용하는 **내 브라우저에만** 저장되며 어디로도 전송되지 않습니다(GitHub에 직접 요청할 때만 사용).
> 단, 토큰을 아는 사람은 저장소를 수정할 수 있으니 **절대 다른 사람에게 알려 주지 마세요.**
> 공용 컴퓨터에서는 쓰지 말고, 유출이 의심되면 GitHub에서 그 토큰을 삭제(Revoke)하면 즉시 무효화됩니다.
> admin.html 페이지 자체는 누가 열어도 토큰이 없으면 아무것도 바꿀 수 없어 안전합니다.

### 연결 후 일상 사용법
- 글 수정: 탭에서 고치면 아래에 "저장 안 된 변경 n건" 바가 떠요 → **[🚀 사이트에 올리기]** → 1~2분 뒤 반영
- 사진 추가: 📚 책 표지나 📸 사진 탭에서 **[📷 올리기]** → 사진 선택 → 자동으로 가로 1100px JPEG로 압축되어
  images/ 폴더에 업로드되고 경로도 자동 입력 → [🚀 사이트에 올리기]
- 활동지 추가: 📄 활동지 탭에서 [＋ 추가] → 프로그램 아이디·버튼 이름 입력 → **[📄 올리기]**로 PDF 업로드 → [🚀 올리기]
  → 사이트의 해당 프로그램에 "📄 내려받기" 버튼이 자동으로 생깁니다
- GitHub 미연결 상태에서 [올리기]를 누르면: 압축된 파일이 컴퓨터로 내려받아지고 경로가 채워지니,
  그 파일을 images/(또는 pdfs/) 폴더에 넣고 기존 방식으로 배포하면 됩니다.

## 1-2. 북트레일러 영상 — 어떤 채널이든 OK 🎬

📚 책 탭의 영상 주소 칸에는 **아무 링크나** 붙여넣을 수 있습니다.

- 비워 두면 "영상이 곧 공개됩니다" 표시가 나옵니다.

## 2. 새 책(강연 프로그램) 추가하기 — 단계별 따라 하기 🐣

새 그림책 『반짝반짝 이 닦기』(예시)가 나왔다고 가정할게요. 15분이면 충분합니다.

**0단계. 준비물**: 표지 사진 1장, 책 소개 문구, 교육 효과·독후활동 아이디어 메모

**1단계. 아이디 정하기**
- 영문 소문자로 짧게. 예: `chikchik`
- 이 아이디 하나로 책 ↔ 강연 ↔ 사진이 서로 연결됩니다. 어딘가 적어 두세요.

**2단계. 표지 사진 넣기**
- 사진을 가로 1100px 이하 JPEG로 줄여서(미리캔버스, 알씨, 맥 미리보기 등 아무 도구나 OK)
- `images/cover-chikchik.jpg` 처럼 저장해 `images/` 폴더에 넣습니다.
- 파일명 규칙: 영문/숫자/하이픈(-)만, 띄어쓰기 금지.

**3단계. admin.html → 📚 책 탭**
- [＋ 새 책 추가] 클릭 → 칸 채우기:
  - 아이디: `chikchik` / 이모지: `🪥` / 책 제목: `반짝반짝 이 닦기`
  - 표지 사진 경로: `images/cover-chikchik.jpg`
  - 구매 링크, 책 소개 입력. 북트레일러가 있으면 영상에 유튜브 주소 붙여넣기(없으면 비워 두기)
- **[💾 books.json 내려받기]**

**4단계. admin.html → 🎤 강연 탭**
- [＋ 새 강연 프로그램 추가] 클릭 (기존 프로그램 내용이 복사되어 오니 고치기만 하면 됩니다):
  - 아이디: `chikchik` / **연결된 책 아이디: `chikchik`** ← 책 탭과 똑같이!
  - 포인트 색 고르기, 슬로건·소개 문장·키워드·교육 효과·추천 대상 수정
  - 진행 순서 4단계, 독후활동 카드들 수정
  - 주제 필터 버튼 문구: `🪥 양치 습관` (비우면 필터 버튼 없음)
  - 추천 이유·추천 키워드: '1분 강연 추천' 결과 화면에 나오는 문구
- **[💾 lectures.json 내려받기]**

**5단계. (사진이 있다면) 📸 사진 탭**
- [＋ 새 사진 추가] → 어느 프로그램: `chikchik`, 경로·설명 입력 → **[💾 gallery.json 내려받기]**

**6단계. (선택) '1분 강연 추천'에 새 주제 버튼 넣기**
- `data/site.json`을 열어 `"reco"` → `"steps"` → 첫 번째 `"options"` 목록에 한 줄 추가:
  ```json
  { "label": "🪥 양치 습관 교육", "target": "chikchik" },
  ```
- 저장 전에 🩺 파일 검사로 확인하면 안심!

**7단계. 교체하고 배포**
- 내려받은 `books.json`, `lectures.json`(+`gallery.json`, `site.json`)을 `data/` 폴더에 덮어쓰기
- 폴더 전체를 Cloudflare Pages에 다시 업로드

끝! 대표 강연 카드, 상세 섹션, 필터 버튼, 강연 추천 결과까지 **전부 자동으로** 생깁니다.

## 3. JSON 파일 항목별 설명서 📖

JSON을 직접 고칠 때 참고하세요. 규칙은 세 가지뿐입니다.
① 글은 곧은따옴표 `"..."` 안에 ② 항목 사이엔 쉼표, **마지막 항목 뒤엔 쉼표 없음** ③ 저장은 UTF-8.
JSON 파일 안에는 주석(설명 글)을 쓸 수 없어서, 아래에 각 줄의 뜻을 풀어 놓았습니다.

### 📚 data/books.json — 책 한 권의 정보
```json
{
  "id": "banchan",
  "emoji": "🍙",
  "title": "판타스틱 반찬 특공대",
  "cover": "images/cover-banchan.jpg",
  "coverAlt": "판타스틱 반찬 특공대 표지",
  "buyUrl": "https://www.yes24.com/product/goods/125184490",
  "buyLabel": "예스24에서 책 보기 →",
  "storyIcon": "📕",
  "story": "편식하는 아이들을 위해 뭉친 반찬 특공대! ...",
  "videos": [
    { "url": "https://www.youtube.com/watch?v=r6PTNtvb39A", "title": "북트레일러", "sub": "한 줄 설명" }
  ]
}
```
| 항목 | 뜻 |
|---|---|
| `id` | 이 책의 이름표(영문 소문자). lectures의 `bookId`, gallery의 `lectureId`와 맞춰야 함 |
| `emoji` | 제목 앞에 붙는 이모지 |
| `title` | 책 제목 |
| `cover` / `coverAlt` | 표지 사진 경로 / 사진 설명(시각장애인용 화면낭독기가 읽는 글) |
| `buyUrl` / `buyLabel` | 구매 링크 주소 / 링크에 보이는 문구 |
| `storyIcon` / `story` | 책 소개 상자의 아이콘 / 소개 글 |
| `videos` | 영상 목록. `url`에 유튜브·인스타그램·네이버TV 등 **아무 채널의 주소**를 붙여넣으면 됩니다(1-2번 참고). 비우면("") "곧 공개됩니다" 표시 |

### 🎤 data/lectures.json — 강연 프로그램 하나의 정보
```json
{
  "id": "banchan",
  "bookId": "banchan",
  "theme": "tomato",
  "eyebrow": "강연 프로그램 ①",
  "tagline": "골고루! 맛있게! 즐겁게! — ...",
  "oneLiner": "대표 강연 카드에 보일 한 문장",
  "keywords": [ "#식습관", "#편식 개선" ],
  "filterLabel": "🍙 식습관",
  "effects": [ "교육 효과 1", "교육 효과 2" ],
  "targets": [ "추천 대상 1", "추천 대상 2" ],
  "flow": [ { "title": "1단계 이름", "desc": "짧은 설명" } ],
  "activitiesTitle": "독후활동 — 읽고, 만들고, 약속해요!",
  "activities": [ { "title": "활동 이름", "desc": "설명 · 준비물" } ],
  "note": { "icon": "🏅", "title": "특별 안내 제목", "text": "내용" },
  "videosTitle": "북트레일러",
  "galleryTitle": "활동 사진",
  "recoWhy": "1분 강연 추천 결과에 나오는 추천 이유",
  "recoKeywords": [ "#식습관" ],
  "recoActs": "대표 독후활동 한 줄"
}
```
| 항목 | 뜻 |
|---|---|
| `id` / `bookId` | 프로그램 이름표 / 어느 책의 강연인지 (books의 `id`와 동일하게). **책이 없는 강연**(창작·출판 등)은 `bookId`를 빼고 대신 `title`(강연 제목)·`emoji`·`intro`(강연 소개)를 씁니다 |
| `category` | 강연 프로그램 탭 분류: `kids`(📚 어린이 그림책) 또는 `creative`(✍️ 창작·출판). 없으면 kids로 표시. admin의 "강연 카테고리"에서 선택하면 홈페이지 해당 탭에 자동 표시됩니다 |
| `theme` | 포인트 색: `tomato`(빨강) `mint`(민트) `leaf`(초록) 중 하나 |
| `filterLabel` | 프로그램 목록 위 주제 필터 버튼 문구. 지우면 버튼이 안 생김 |
| `flow` / `activities` | 진행 순서 / 독후활동 카드. 번호는 자동으로 붙습니다 |
| `note` | 노란 점선 안내 상자(임명장 수여식 같은 것). 필요 없으면 항목째 지워도 됨 |
| `recoWhy` `recoKeywords` `recoActs` | '1분 강연 추천'의 결과 카드에 나오는 문구들 |

### 📸 data/gallery.json — 활동 사진 한 장
```json
{ "lectureId": "banchan", "image": "images/gallery-banchan-9.jpg", "caption": "OO도서관 강연", "alt": "OO도서관 강연 모습" }
```
`lectureId` = 어느 프로그램의 갤러리에 넣을지 / `image` = 사진 경로 / `caption` = 폴라로이드 아래 글 / `alt` = 화면낭독기용 설명

### 📰 data/news.json — 소식 하나
```json
{ "date": "2026", "icon": "🎉", "title": "소식 제목", "text": "한두 문장 설명", "url": "" }
```
`url`을 넣으면 카드 클릭 시 그 주소로 이동, 비우면 링크 없는 카드. **맨 위에 추가한 소식이 가장 먼저** 보입니다.

### 💬 data/reviews.json — 후기 하나
```json
{ "text": "후기 내용", "author": "OO초등학교 교사", "context": "판타스틱 반찬 특공대 강연" }
```
> ⚠️ 처음 들어 있는 후기들은 예시(`"sample": true` 표시)입니다. 실제 후기로 교체해 주세요.

### ❓ data/faq.json — 질문 하나
```json
{ "q": "질문", "a": "답변" }
```

### 📄 data/pdfs.json — 독후활동 PDF 하나
```json
{ "lectureId": "banchan", "title": "식판 활동지", "file": "pdfs/banchan-sikpan.pdf" }
```
`lectureId` = 어느 프로그램에 버튼을 만들지 / `title` = 버튼에 보이는 이름 / `file` = pdfs 폴더 속 PDF 경로.
사이트의 해당 프로그램 섹션에 "활동지 내려받기" 버튼이 자동으로 생깁니다. PDF 파일 자체는 `pdfs/` 폴더에 넣습니다(admin에서 [📄 올리기] 사용 가능).

### ⚙️ data/site.json — 한 번만 나오는 내용 전부
히어로 문구, 메뉴, WHY 카드 4장, 강연 추천 질문·버튼, 작가 소개, 연락처, 푸터가 들어 있습니다.
자주 고치는 항목은 admin.html의 [⚙️ 기본 정보] 탭에서 편하게 고칠 수 있고,
나머지는 파일에서 한글 문구를 찾아 따옴표 안의 글자만 바꾸면 됩니다.
- 이름이 `titleHtml`처럼 **Html로 끝나는 항목**만 `<br>`(줄바꿈), `<span class="hl">강조</span>` 태그 사용 가능
- 연락처(`contact.contacts`)는 보이는 글자(`label`)와 연결 주소(`href`)를 함께 수정
  (전화 `tel:010-…` / 메일 `mailto:…` / 인스타그램은 전체 주소)

### 섹션 통째로 숨기기
목록형 파일(news, reviews, faq, gallery)의 내용을 다 지우고 대괄호만 `[]` 남기면
그 섹션과 메뉴가 조용히 사라집니다. 오류는 나지 않습니다.

## 4. 문법이 틀렸을 때 — 이렇게 알 수 있어요 🚨

- **사이트 화면 위에 검은 안내 띠**가 뜹니다: 어느 파일, 몇 번째 줄 근처, 무엇을 확인할지 알려 줍니다.
  (문제가 없으면 절대 나타나지 않아요. 띠가 떠도 사이트의 나머지 부분은 정상 동작합니다.)
- **admin.html → 🩺 파일 검사**에서 파일을 열면 잘못된 줄을 ▲ 표시로 정확히 짚어 줍니다.
- 자주 나는 실수 3가지:
  1. 마지막 항목 뒤에 쉼표를 남김 → 그 쉼표 삭제
  2. 한글 입력기 때문에 곧은따옴표(")가 둥근따옴표(“)로 바뀜 → "로 교체
  3. 항목 사이 쉼표 빠뜨림 → `}` 와 `{` 사이에 쉼표 추가

## 5. 내 컴퓨터에서 미리 보기

`index.html`을 더블클릭으로 바로 열면 브라우저 보안 정책 때문에 JSON을 읽지 못합니다.
- **VS Code + Live Server 확장**(추천): index.html 우클릭 → "Open with Live Server"
- 또는 이 폴더에서 터미널 열고 `python -m http.server` → 브라우저에서 `http://localhost:8000`
- admin.html도 같은 방식으로 열면 데이터가 자동으로 불러와집니다.
  (더블클릭으로 연 경우엔 각 탭에서 파일을 직접 선택해 불러올 수 있어요.)

## 6. 배포하기

**Cloudflare Pages** (지금 쓰는 곳): 대시보드 → 프로젝트 → 새 배포 만들기 → **이 폴더 전체를 드래그**해서 업로드. 빌드 설정 불필요.

**GitHub Pages**: 저장소 루트에 이 폴더의 파일을 전부 올리고 → Settings → Pages → Branch `main`(root) 선택. 1~2분 뒤 `https://아이디.github.io/저장소이름/`

둘 다 별도 서버가 전혀 필요 없습니다.

## 7. 문제 해결 표

| 증상 | 해결 |
|---|---|
| 화면 위에 검은 오류 띠 | 띠에 적힌 파일을 🩺 파일 검사로 열어 고치기 |
| 섹션 하나가 통째로 안 보임 | 그 파일이 비어 있거나(`[]`) 문법 오류. 위와 동일 |
| "데이터를 불러오지 못했어요" 전체 화면 | site.json 문제이거나 더블클릭으로 연 경우(5번 참고) |
| 사진이 안 나옴 | 경로·파일명이 실제 파일과 다름(대소문자까지 동일해야 함) |
| 새로고침해도 그대로 | Ctrl+Shift+R (강력 새로고침) |
| admin에서 고쳤는데 사이트 그대로 | GitHub 연결 시 [🚀 사이트에 올리기]까지 눌렀는지, 미연결 시 [내려받기]→교체→재배포까지 했는지 확인 |
| **활동지(PDF) 버튼이 사이트에 안 보임** | 관리 페이지는 data와 pdfs 파일만 올리고 **화면을 그리는 프로그램 파일은 안 바꿉니다.** 저장소의 `index.html` `js/app.js` `css/style.css` `admin.html` 4개가 최신 버전인지 확인하고, 아니면 GitHub 저장소 화면에서 Add file → Upload files로 4개를 덮어써 주세요. 그래도 안 보이면 F12 콘솔에 lectureId 연결 안내가 찍혀 있는지 확인하세요 |
